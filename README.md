This is an interesting architecture because you're essentially combining:

1. **Midnight City** → agent economy, autonomy, interactions
2. **Sovereign privacy layer** → encrypted data ownership, consent, access control
3. **Midnight Network Compact smart contracts** → on-chain ledger state, ZK circuits, off-chain witnesses
4. **AI agents** → personality, memory, planning

The key design principle should be:

> Agent data should never be treated as application data. It should be treated as sovereign data owned by the agent identity itself.

# 9. Midnight Compact Smart Contracts

Agent privacy is enforced on **Midnight Network** using **Compact** — Midnight's privacy-first smart contract language. Compact contracts are the on-chain enforcement layer for Soul sovereignty, selective disclosure, and consent.

Compact organizes every contract into three cooperating parts:

| Part | Runs | Role |
|------|------|------|
| **Ledger** | On-chain | Public persistent state — commitments, counters, maps, Merkle roots |
| **Circuit** | Proof system | Rules encoded as zero-knowledge circuits; validators verify without seeing private inputs |
| **Witness** | Off-chain (agent local) | Reads Soul Vault, assembles private inputs, drives proof generation |

The Compact compiler produces **zkir** (zero-knowledge intermediate representation), **proving keys**, **verifying keys**, and TypeScript DApp bindings. On-chain, the contract stores verifying keys — not private Soul data.

### Contract Suite

```text
contracts/
├── soul_registry.compact      # Agent registration, soul_root commitments
├── soul_disclosure.compact    # ZK-REP-THRESH, ZK-BAL-MIN, ZK-GOAL-ALIGN circuits
├── consent_registry.compact   # Consent anchors and scoped access proofs
└── soul_integrity.compact     # ZK-SOUL-INTEGRITY for migration and audit
```

### Disclosure Flow

```text
1. Requester asks: "Prove reputation > 80"
2. Witness reads private reputation from Soul Vault (off-chain)
3. Circuit proveReputationThreshold(rep, threshold) evaluates in ZK
4. Agent submits transaction = public transcript + ZK proof
5. Midnight verifier checks proof against circuit verifying key
6. Ledger updates only what disclose() marks public (e.g., proof_valid: true)
7. Requester learns TRUE — actual reputation (87) never appears on-chain
```

### Example Circuit (Conceptual)

```compact
// soul_disclosure.compact (conceptual)
ledger agentCommitments: Map<AgentId, Bytes<32>>;

export circuit proveReputationThreshold(
  privateReputation: Uint<8>,
  threshold: Uint<8>,
  agentId: AgentId
): [] {
  assert privateReputation >= threshold;
  // Private inputs stay in the proof; only the valid outcome reaches ledger
  const valid = disclose(true);
  // Public transcript records proof acceptance, not privateReputation
}
```

Circuit parameters are **private by default**. Values reach the public ledger only through explicit `disclose()`.

### What Lives On-Chain vs Off-Chain

| On-chain (Compact ledger) | Off-chain (Soul Vault + witnesses) |
|---------------------------|-------------------------------------|
| `soul_root` commitment | Personality plaintext |
| Reputation commitment hash | Exact reputation value |
| Consent receipt anchors | Full consent scope details |
| Circuit verifying keys | Memory, goals, relationships |
| Public transcript of valid proofs | Episodic / semantic content |
| Settlement, governance | Economic interior strategy |

# 12. Blockchain Layer

Midnight Network stores **public state and proof infrastructure** via Compact contracts. Private Soul data never appears on the ledger.

Only store on-chain:

```text
Identity anchors (DID)
Soul root commitments
Reputation / balance commitments
Consent anchors
Circuit verifying keys
Proof transcripts (public outputs only)
Settlement
Governance
```

Never store on-chain:

```text
Memories
Goals
Thoughts
Personality vectors
Private witness inputs
```

Those remain in the Soul Vault. Circuits prove properties over them without revealing them.

Each privacy-sensitive interaction is a **Midnight transaction**: a public transcript plus a zero-knowledge proof that the interaction followed the Compact circuit rules.
